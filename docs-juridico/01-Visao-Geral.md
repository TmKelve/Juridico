# Visão Geral do SaaS Jurídico

## 1. Propósito do produto
Criar uma plataforma jurídica focada em experiência de trabalho, não em módulos isolados.

- Dashboard inicial por perfil
- Painel de responsabilidades pessoais
- Menus adaptados por função
- Permissões granulares + escopo de dados
- Metas: reduzir complexidade, aumentar ação e escala comercial

## 2. Horizonte proposto
Fase 1 (MVP): Advogado, Estagiário, Financeiro, Administrativo, Atendimento
Fase 2: Sócio, Cliente Externo, Operações avançadas (OCR, IA, autotriagem)

## 3. Camadas de plataforma
1. Núcleo operacional: processos, prazos, documentos, tarefas.
2. Núcleo de gestão: financeiro, gestão do escritório, relatórios.
3. Núcleo de crescimento: CRM, portal do cliente, integrações.

## 4. Regras de ouro de UX
- Toda home responde: o que aconteceu, o que exige ação, o que merece atenção.
- A home é acionável (não somente informativa).
- Menu lateral reduzido por perfil; escolher atalhos certos.
- Escopo sempre vence perfil.
- Exclusão arquiva; dados sensíveis trilha de auditoria.

## 5. Painel de Responsabilidades
Usuário vê primeiro (campo de batalha):
- o que está sob responsabilidade dele
- atrasados
- vence hoje
- aguardando resposta dele
- recém atribuído
- dependências de outros setores
